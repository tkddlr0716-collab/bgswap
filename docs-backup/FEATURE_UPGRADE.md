## 현재 상태 분석

### 유료로 받는 것
| 항목 | 무료 | Starter $4.99 | Pro $29 |
|------|------|---------------|---------|
| 상품 수 | 1 | 10 | 100 |
| 해상도 | 512px | 1024px | 1024px |
| 워터마크 | 있음 | 없음 | 없음 |
| 배경 수 | 1 (흰색) | 5 | 5 |
| 드롭섀도우 | 없음 | 없음 (제거됨) | 없음 (제거됨) |
| ZIP 다운로드 | 없음 | 있음 | 있음 |

### 문제점: "돈 내고 뭘 받는 건데?"

1. **배경 5개가 고정** — 흰색/회색/웜/쿨/다크만. 내가 원하는 색을 못 고름
2. **출력 크기가 1024px 고정** — Amazon은 2000px 권장, Etsy도 최소 2000px. 1024px은 부족
3. **상품 1개당 사진 1장만** — 실제 셀러는 한 상품을 여러 각도로 찍음
4. **결과물 커스터마이징 불가** — 그림자 ON/OFF, 여백 조절 등 아무것도 못 함
5. **프리뷰가 너무 허접** — 512px 워터마크 이미지 1장으로는 품질 판단 어려움
6. **재생성 불가** — 결과가 마음에 안 들면 돈 내고도 다시 못 함

## 개선 우선순위

### Tier 1: 즉시 체감되는 개선 (3~4일 + 테스트 반일)

#### 1-1. 출력 해상도 2048px 업그레이드
- **현재**: 1024px 고정
- **변경**: 유료 플랜 2048px (Amazon/Etsy/Shopify 모두 충족)
- **작업**: `compositor.ts`에서 paid mode의 size를 2048로 변경

**비용 영향 분석:**

| 항목 | 1024px (현재) | 2048px (변경) | 비고 |
|------|-------------|-------------|------|
| JPEG 파일 크기 (1장) | ~60~80KB | ~200~300KB | 약 3~4배 |
| 5배경 ZIP 크기 | ~400KB | ~1.2~1.5MB | 모바일 다운로드 1~2초 |
| R2 저장 (100상품) | ~40MB | ~150MB | R2 무료 10GB 이내 |
| R2 월비용 (1000유저) | $0.15 | $0.50 | 무시 가능 |
| Sharp 메모리 | ~100MB | ~400MB | **Vercel 한도 확인 필수** |
| Sharp 처리 시간 | ~0.5초/장 | ~1.5초/장 | 5장 = 7.5초 |

**Vercel 메모리 한도:**
- Free/Hobby: 1024MB
- 2048px Sharp 합성 5장 = ~400MB peak → 1024MB 이내 가능
- 단, 동시 요청 시 OOM 가능 → **테스트 필수**
- OOM 발생 시 대안: 순차 처리 (5장을 한번에 메모리에 안 올림)

#### 1-2. 커스텀 배경색 선택
- **현재**: 5가지 고정 프리셋
- **변경**: 5 프리셋 유지 + 커스텀 HEX 색상 1개 추가 (유료만)
- **UI**: result 페이지에 색상 피커 (유료 결제 후에만 활성화)

**입력 검증 (보안):**
```typescript
function validateHexColor(input: string): { r: number; g: number; b: number } | null {
  // 정규식으로 엄격 검증
  const match = input.match(/^#?([0-9a-fA-F]{6})$/);
  if (!match) return null;
  const hex = match[1];
  return {
    r: parseInt(hex.substring(0, 2), 16),
    g: parseInt(hex.substring(2, 4), 16),
    b: parseInt(hex.substring(4, 6), 16),
  };
}
// API에서: 검증 실패 시 400 반환, 빈 문자열/undefined는 무시 (기본 5색만)
```

**무료 사용자 UX:**
- 색상 피커는 보이지만 비활성화 (회색 잠금 아이콘)
- 클릭 시 "Upgrade to Starter to use custom colors" 안내
- 강제 차단이 아닌 업셀 유도

#### 1-3. 무료 프리뷰 5배경 썸네일
- **현재**: 512px + 워터마크 + 흰 배경 1장
- **변경**: **256px 워터마크 5배경 썸네일** → 결과 페이지에서 격자로 표시

**남용 방지 설계:**

| 항목 | 현재 | 변경 후 |
|------|------|---------|
| 무료 1회당 합성 수 | 1장 (512px) | 5장 (256px) |
| 1장당 처리 비용 | 256px 기준 ~0.1초 | 256px 기준 ~0.1초 |
| 1회 총 처리 시간 | ~3초 (배경제거+합성) | ~4초 (배경제거+합성5장) |
| 일일 캡 100건 시 합성 수 | 100장 | 500장 |
| 서버 부하 체감 | 기준 | ~1.3배 (합성 자체는 가벼움, 배경 제거가 비용의 95%) |

**결론:** 배경 제거가 비용의 대부분이고 이건 1회만 하므로, 256px 합성 4장 추가는 부하 미미. 일일 캡 100건 유지해도 문제 없음.

### Tier 2: 전환율 직접 영향 (3~5일)

#### 2-1. 그림자 ON/OFF 옵션
- **현재**: 그림자 완전 제거 상태
- **변경**: 유료 결제 후 "Add drop shadow" 토글 → 접촉 그림자 추가된 버전 재생성
- **UI**: 다운로드 페이지에서 "Shadow: ON/OFF" 토글
- **작업**: compositor에 shadow 파라미터 복원 (개선된 접촉 그림자), 재생성 API 추가

#### 2-2. 여백(패딩) 조절
- **현재**: 제품이 프레임의 80% 차지, 고정
- **변경**: 슬라이더로 60%~95% 조절 가능 (유료만)
- **이유**: Amazon은 85%+ 채우기 권장, 웹사이트용은 여백 많이 원함

#### 2-3. 마켓플레이스별 프리셋 출력
- **현재**: 정사각 1024px 고정
- **변경**: 마켓플레이스 규격 버튼 클릭

| 마켓플레이스 | 크기 | 비율 | 비고 |
|-------------|------|------|------|
| Amazon Main | 2000×2000 | 1:1 | 흰 배경 필수 |
| Amazon Secondary | 2000×2000 | 1:1 | 라이프스타일 허용 |
| Etsy | 2000×2000 | 1:1 | |
| Shopify | 2048×2048 | 1:1 | |
| eBay | 1600×1600 | 1:1 | |
| Instagram | 1080×1080 | 1:1 | |
| Custom | 사용자 입력 | 사용자 입력 | **정사각형만 지원 (v1)** |

> v1에서는 정사각형만 지원. 비정사각 비율(예: 4:3, 16:9)은 compositor의 캔버스 로직 변경이 필요하므로 Phase C로 연기.

---

### Tier 3: 차별화 기능 (1~2주)

#### 3-1. 배치 업로드 + 결과 갤러리
- **현재**: 최대 5장 업로드, 첫 번째 사진만 처리
- **변경**: 5장 모두 처리 → 각각 5배경 = 최대 25장

**DB 호환성:**
- 현재 images 테이블: `id, order_id, r2_key, type, resolution, style, expires_at`
- order 1개에 image N개 관계 → **스키마 변경 불필요**
- 단, 현재 generate API가 `LIMIT 1`로 첫 이미지만 처리 → 반복 처리 로직 추가 필요

#### 3-2. AI 이미지 보정 (업스케일링)
- **변경**: 저해상도 폰카 → 고해상도 (Real-ESRGAN via Replicate)

**비용/시간 영향:**
- Replicate Real-ESRGAN: ~$0.01/이미지, ~5초 추가
- 전체 플로우: 배경 제거(5초) + 업스케일(5초) + 합성(7.5초) = ~17.5초
- "10초" 마케팅 메시지와 충돌 → **선택적 옵션**으로 제공, 기본 OFF
- "HD Enhance" 토글 추가, 처리 시간 안내: "Takes ~20 seconds for enhanced quality"

#### 3-3. 배경 템플릿 (단색 외)
- 그라디언트, 대리석, 나무결, 패브릭 등 10~20개 템플릿
- 구현: Sharp composite에 텍스처 이미지 레이어 추가
- public/textures/ 에 텍스처 이미지 저장

## 구현 권장 순서

```
Phase A (3~4일 + 테스트)
├── 1-1. 출력 해상도 2048px + Vercel 메모리 테스트
├── 1-2. 커스텀 배경색 (HEX 검증 + UI + 무료 UX)
└── 1-3. 무료 프리뷰 5배경 썸네일

Phase B (3~5일)
├── 2-1. 그림자 ON/OFF
├── 2-2. 여백 조절
└── 2-3. 마켓플레이스 프리셋 (정사각형 v1)

Phase C (1~2주)
├── 3-1. ✅ 배치 결과 갤러리 (상품별 탭 UI, 프로그레스 바 수정)
├── 3-2. 보류 — AI 업스케일링 (비용/속도 영향, 트래픽 확보 후)
└── 3-3. ✅ 배경 템플릿 (SVG 텍스처 5종: Marble/Wood/Linen/Concrete/Paper)
```

## 경쟁사 대비 가치 비교 (Phase A 완료 후)

| 기능 | remove.bg | PhotoRoom | BgSwap (현재) | BgSwap (Phase A) |
|------|-----------|-----------|---------------|------------------|
| 배경 제거 | ✅ | ✅ | ✅ | ✅ |
| 배경 교체 | ❌ | ✅ (AI) | ✅ (5 프리셋) | ✅ (5+커스텀) |
| 출력 해상도 | 최대 25MP | HD (유료) | 1024px | **2048px** |
| 커스텀 색상 | ❌ | ✅ | ❌ | **✅** |
| 무료 미리보기 | 저해상도 1장 | 제한적 | 512px 1장 | **5배경 썸네일** |
| 구독 | 선택 | 필수 | ❌ | ❌ |
| 단가 (100장) | $0.15~0.43 | $12.99/mo | $0.29 | $0.29 |

> remove.bg 최대 해상도가 25MP(~5000×5000)이므로 2048px은 여전히 낮지만, remove.bg는 배경 제거만 하고 교체는 안 함. BgSwap은 "제거+교체+5배경 자동" 패키지이므로 직접 비교 대상이 다름.

## 결론

**Phase A만 하면:**
- 2048px 고해상도 → "마켓플레이스 규격 충족" (현재 1024px은 부족)
- 커스텀 색상 → "내 브랜드에 맞춘 배경" (remove.bg에 없는 기능)
- 5배경 프리뷰 → "다 보고 결제" (전환율 직접 영향)

이 3개만 추가하면 $5/$29 가격이 납득 가능해짐.
